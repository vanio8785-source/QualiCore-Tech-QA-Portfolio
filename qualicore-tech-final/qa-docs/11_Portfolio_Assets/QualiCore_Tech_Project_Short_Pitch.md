# Project Short Pitch
## QualiCore Tech — Manual QA Portfolio Project

---

## 30-Second Pitch (verbal / phone screening)

"QualiCore Tech is a manual QA portfolio project where I tested a realistic React e-commerce application end-to-end. I designed the full QA documentation suite — test plan, user stories, 35 test cases, RTM — executed everything, found 15 bugs, and wrote up a test summary report. It covers search, filtering, sorting, cart, wishlist, auth, and checkout. The app has intentional defects built in, and I found all of them plus a few extras during exploratory testing."

---

## Elevator Pitch (networking / recruiter call)

"I built a QA portfolio project called QualiCore Tech. It's a demo e-commerce app that I used to practice the full manual testing lifecycle — from reviewing requirements to writing test cases, executing them, and reporting bugs.

I found 15 defects, including a cart subtotal calculation error, a sorting bug that affected a whole product category, and several validation gaps in the checkout flow. I documented everything in professional format — test plan, RTM, bug reports, execution log, and summary report.

The whole package is on GitHub, and the app is live on Vercel so anyone can interact with it. I can walk you through any of the documents if you'd like to see how I approach QA."

---

## Written Pitch (email / application cover letter paragraph)

"One of the projects I'm most proud of in my portfolio is QualiCore Tech — a full manual QA cycle I conducted on a React e-commerce demo application. I produced the complete QA documentation suite from scratch, including a test plan, 50 user stories, 35 detailed test cases, and a Requirements Traceability Matrix. I executed all test cases, identified 15 reproducible defects across functional areas including cart calculations, form validation, sorting logic, and data consistency, and delivered a test summary report with quality assessment and release recommendations. This project demonstrates my ability to work through the full testing lifecycle systematically and produce professional documentation that a QA team or hiring manager can evaluate directly."
